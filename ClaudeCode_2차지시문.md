# ClaudeCode 2차 작업 지시문 (2026-06-26)

> **수신**: Claude Code (노트북, 개발자)
> **발신**: 채팅 Claude (기획자) ← 이주혁(고객) 우선순위 확정 반영
> **전제**: 1차 진단 보고(YELLOW) 검수 완료. 진단은 신뢰함. 아래는 그 위에 쌓는 후속 작업.
> **기준**: 전문적·엄격. 추정 금지. 부분동작을 완성으로 착각 금지. 토스 실주문 절대 금지.

## 1순위 — 보안 마무리 (최우선, 다른 작업보다 먼저)

1차에서 봇 토큰 로그 누출을 마스킹·정리한 것은 확인했다. 그러나 두 가지가 미결이다.

### 1-1. 토큰 재발급 (이주혁 직접 — 너는 코드만 준비)
- **이주혁님이 BotFather에서 봇 토큰을 재발급**한다. 이건 사람이 한다.
- 너는 `.env`의 `TELEGRAM_BOT_TOKEN` 한 줄만 교체하면 동작하도록 코드 경로를 확인·보고하라.
- 재발급 전후로 하드코딩된 토큰이 코드 어디에도 없는지 재확인하라.

### 1-2. 전송경로 마스킹 점검 (★ 1차에서 빠진 부분)
- `_redact`가 `log()`에만 적용됐는데, **토큰이 텔레그램 전송 본문·다른 print·예외 메시지로
  새는 경로가 더 있는지** 전수 점검하라.
- 점검 대상: `send_telegram`, 모든 `print(...)`에 토큰/URL이 들어갈 수 있는 곳,
  예외 핸들러에서 `requests` 예외 메시지를 출력·전송하는 모든 지점.
- 토큰이 들어갈 수 있는 모든 출력 경로에 `_redact`를 적용하라.
- 점검 결과를 "경로 목록 + 적용 여부" 표로 보고하라.

### 1-3. 보고
- 위 1-1, 1-2 결과 + `.env 외 토큰 비밀부 0건` 재확인을 보고하라.

## 2순위 — 한국 브리핑(R9) + 미국 4기둥 미래성장성★(R15)

### 2-1. R9 한국 브리핑 6항목 (현재 ⬜)
CLAUDE.md 4.2절 명세대로 "오늘 한국시장 어때?" 트리거 시 6항목 출력:
1. 주요지수&환율: 코스피, 코스닥, USD/KRW, 외국인 순매수/매도
2. 코스피 대장주: 삼성전자, SK하이닉스, 현대차, 기아, 네이버, 카카오 (현재가+등락)
3. 로봇관련주: 두산로보틱스, 레인보우로보틱스, HD현대로보틱스 동향
4. 저평가 가치주 레이더: PBR/PER 기준 + 근거
5. 핵심뉴스 3개: 증시영향 + 방향성
6. 리스크 & 단기전망: 근거 포함
- 데이터는 네이버 API 우선. 수집 실패 시 R7 원칙(GO/출력 보류 + 실패 알림) 적용.
- 전문적 어조, 실시간 지표, 근거 필수.

### 2-2. R15 미래성장성★ (현재 🔶 — 밸류·타이밍만 있음)
CLAUDE.md 3.1절 4기둥 중 **미래성장성★(최우선 기둥)**이 비어 있다. 정량/정성 로직 추가:
- 실적 추세, 가이던스, 신사업, 메가트렌드, 회사 분위기를 평가에 반영
- 단, 정성 항목은 무리하게 수치화하지 말 것. 근거 텍스트 + 신뢰도 표기 수준으로 시작.
- 펀더멘털(매출·EPS 성장·영업이익률·FCF·부채·자사주/배당)도 함께 보강하면 R15가 ✅에 근접.

### 2-3. 보고
- R9·R15 구현 위치(파일:함수) + 실제 출력 샘플 1회 + self_check 재실행 결과.

## 3순위 — 감시 종목 16개 리스트 코드 확인 (★ 1차 보고 누락)

1차 보고에 **감시 대상 종목 리스트의 실제 내용**이 없었다. R13(위험)·R14(기회)·R19(불타기)가
반쪽이 되지 않으려면 이게 코드에 정의돼 있어야 한다.

- 코드에서 감시 종목 리스트(한국/미국 각각)를 찾아 **실제 종목 전체를 그대로 보고**하라.
- 현재 보유 16종목과 대조: MU, AMD, AAPL, GOOGL(A+C), TSLA, TSM, NFLX, AMZN, CAT, LIT, RKLB 외.
- 누락된 종목이 있으면 목록화하고, 리스트를 16종목과 일치시킬지 이주혁 판단을 받도록 보고하라.
- 한국 단타 감시 종목과 미국 보유 종목이 코드에서 **트랙별로 분리**돼 있는지 확인하라
  (두 트랙 혼동 금지 — CLAUDE.md 1장).

## 4순위 — 양도세 추적 (R20, 현재 🔶 정적 텍스트만)

> 전제: 이주혁님은 "2026 추가 매도 자제" 원칙 유지 중. 따라서 자동 차단은 급하지 않으나,
> 전제가 흔들릴 때를 대비해 추적 골격은 만들어 둔다.

- 미국 주식 실현차익 누계를 추적하는 구조 설계 (2026 누계 ≈238만원에서 시작, 한도 250만원).
- 매도 시 잔여 한도 경고 출력 (예: "잔여 한도 ≈12만원, 추가 매도 시 과세 발생").
- **실제 차단은 알림 수준으로 시작.** 자동으로 매도를 막는 로직은 이주혁 승인 후.
- 한도 리셋(2027-01-01) 반영.
