## 0. 문제 정의 (관측된 사실)
- 봇 가치주판정 송신에서 **NFLX 현재가 74달러 / 200일선 97달러**로 나옴.
- 실제 넷플릭스는 **수백 달러대(700~1200)** → 74달러는 명백한 이상치.
- 토스 정렬버그·삼성 5배차와 같은 종류의 데이터 오염 신호. 차트경고(현재가<200일선=하락추세)가 이 틀린 절대가를 근거로 쓰면 판정근거 오염.
- 출처는 yfinance `.info` 또는 차트 데이터로 추정.

## 2. 진단 절차 (가설을 응답 원본으로 하나씩 배제 — 추정 금지)

### [가설 A] yfinance가 NFLX를 다른 종목/심볼로 해석?
- `yfinance.Ticker("NFLX").info`의 `longName`/`shortName`/`symbol` 확인 → 진짜 "Netflix Inc / NFLX"인가?
- `currentPrice`/`regularMarketPrice`/`previousClose` 등 **가격 필드별 값**을 전부 출력(어느 필드가 74고 어느 필드가 정상인지).

### [가설 B] 어느 가격 필드를 쓰는지 — 코드가 잘못된 필드를 읽나?
- `assess_valuation`(또는 차트경고)이 NFLX 현재가/200일선을 **어느 소스·어느 필드**에서 가져오는지 확인.
- `.info`의 여러 가격 필드 중 74가 나온 필드와, 정상값(수백 달러)이 있는 필드를 비교.
- 200일선(97)은 어떻게 계산되나 — `.info`의 `twoHundredDayAverage` 필드인가, 직접 계산인가?

### [가설 C] 일봉/차트 데이터로 교차 확인
- `yfinance.Ticker("NFLX").history(period="5d")`로 최근 종가를 받아 **실제 NFLX 현재가가 얼마인지** 확인(수백 달러대 맞는지).
- `.info` 필드값 vs `.history` 종가 비교 → 어디서 74가 나오는지 격리.

### [가설 D] 분할/통화/조정 이슈?
- NFLX 액면분할 이력, 통화(USD 맞는지) 확인. (NFLX는 분할 이력이 제한적이라 가능성 낮으나 배제용.)

## 3. 보유종목 전체 yfinance 절대가 일괄점검 (NFLX만 문제냐, 전반이냐)
- US_PORTFOLIO/감시종목(MU·AMD·AAPL·GOOGL·TSLA·TSM·NFLX·AMZN·CAT·LIT 등)에 대해 yfinance 현재가를 일괄 조회.
- 각 종목의 yfinance 값이 **상식적 범위인지** 점검(가능하면 알려진 대략값과 비교, 또는 `.info` 현재가 vs `.history` 최근종가 일치 여부로 이상치 자동 탐지).
- **이상치가 NFLX만인지, 다른 종목도 있는지** 표로 정리. 이게 "NFLX 고유 문제 vs yfinance 전반 문제"를 가른다.

## 4. 원인 확정 후에만 수정 제안
- A~D로 원인 확정되면 그에 맞는 **최소 수정 방향만 제안**(이번엔 진단+제안까지, 실제 수정은 원인 명확할 때만).
- 예: 잘못된 가격 필드 사용 → 올바른 필드(`.history` 종가 또는 정확한 `.info` 필드)로 교체. 분할조정 이슈 → `auto_adjust` 설정 점검.
- 원인 불명확하면 수정하지 말고 "미규명" 보고. 추정으로 고치지 마라.
