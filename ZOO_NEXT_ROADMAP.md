## 📋 향후 할 일 (순서)

### A. [지금~기숙사 복귀 전] 그램에서 데이터 축적
- 자동수집 가동(위 검수 통과 후). 그램 켜져 있을 때마다 뉴스+E5 룰베이스 누적.
- 목표: 데스크탑 LLM으로 처리할 "뉴스 재고" 미리 쌓기.
- 부수: 룰베이스 rule_label이 실제로 어떻게 찍히는지 관찰(나중 효용 판단 근거).

### B. [기숙사 복귀 후] 데스크탑 서버화 ★별도 새 대화 "ZOO 서버화"
선결: 데스크탑 물리 접근 필요(현재 기숙사). 본가에선 불가.
1. 데스크탑 포맷 → 서버 OS 세팅(상시 온, 베란다, 전기료 무료 환경).
2. 뉴스 아카이브 저장소를 TB 드라이브에 구축 (= ③데이터센터).
3. 그램이 모은 뉴스 JSONL을 데스크탑으로 이관(파일 복사).
4. 두 노드 데이터 공유 방식 설계 — 권장 후보: 데스크탑=Ollama LLM 서버만, 처리로직 한쪽(동기화 충돌 회피). 대안: GitHub 동기화/공유폴더. ★서버화 때 확정.
5. 역할 확정: 그램=수집·실시간·메인 / 데스크탑=아카이브 보관 + LLM 배치처리.

### C. [서버화 후] 로컬 LLM "무료 시험대" ★별도 새 대화 "ZOO 로컬LLM"
- 데스크탑에 Ollama + 7B급 모델(GTX 1060 6GB 한계 내, 4bit 양자화). 한국어 뉴스 강한 모델 선정.
- E5 2차를 로컬 LLM에 연결. 기존 E5 구조(우선순위 큐+이월)가 "데스크탑이 쌓인 큐를 상시 소화" 구조와 그대로 맞음.
- rule_label vs 로컬 LLM stance 불일치율 로깅(이미 구현된 인프라) → 룰베이스 대비 LLM 효용 무료 검증.
- 데스크탑은 느려도 됨(실시간 아님, 배치). GTX 1060 속도 한계는 설계상 흡수.

### D. [효용 입증 후] 모델 승격 판단
- 로컬 7B로 충분 → 무료 유지.
- 더 높은 정밀도 필요 → 앤트로픽 유료 전환. .env에 ANTHROPIC_API_KEY 한 줄(코드 재작업 0). 월 수 달러 + 캐시 + 일50회 예산 + 콘솔 지출한도.

### E. [병행 가능] 정리·보강 항목 (급하지 않음)
- FMP 402 로그 정리: 무료플랜 뉴스 미지원 확정 → 시도 스킵하고 sources_failed 조용히 기록.
- Untracked 파일 2개(CC_HANDOFF_PROMPT.md, SECURITY_PHASE1_SPEC.md) docs 정리.
- GitHub Actions 자동화(보안 1단계): 그램 꺼져도 수집되게 클라우드 스케줄. 토스 IP 화이트리스트 문제도 고정IP로 동시 해결(메모리 기록).
- 텔레그램 "시장현황" 온디맨드 버튼(미구현).

### F. [별도 트랙] E5 다음 — ⓑ E5 점수화 엔진의 미래성장성 심화
