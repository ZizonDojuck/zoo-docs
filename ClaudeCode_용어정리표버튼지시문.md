## 1. 먼저 현재 봇 버튼 구조 파악 (추측 금지)

봇이 버튼을 어떻게 처리하는지 **실제 코드로 확인**:

- 기존 버튼들(정기 브리핑 등)이 눌리면 어떻게 실행되나? — **콜백 핸들러가 함수를 직접 부르는 구조(경우 A)** 인가, **스케줄 항목과 연결돼야만 하는 구조(경우 B)** 인가?
- 이 파악 결과를 보고에 명시.

## 2. 버튼 연결 (A 우선, 안 되면 B)

### 경우 A — 버튼→함수 직결 가능 (선호)
- "용어정리표" 버튼 1개 추가(기존 버튼 메뉴에 자연스럽게).
- 콜백 핸들러에 분기 추가: 이 버튼이 눌리면 `format_glossary()` 결과를 텔레그램 메시지로 전송.
- 텍스트가 길면 텔레그램 4096자 제한 확인(용어표는 짧아 문제없을 것이나 확인).

### 경우 B — 버튼이 반드시 스케줄과 연결돼야 하는 구조
- 이주혁 아이디어대로: 용어정리표용 스케줄 항목을 **먼 미래 날짜(예: 2999-01-01)** 로 등록 → 자동 실행은 사실상 안 됨(실제 스케줄과 충돌 없음) → 버튼만 활성화.
- 버튼 누르면 그 항목이 실행되며 `format_glossary()` 출력.

### 공통
- **출력은 버튼 누를 때만**(정기 브리핑에 자동 삽입 안 함 — 이주혁: 매번 말고 필요할 때만).
- 기존 버튼·스케줄 동작 **건드리지 말 것**(추가만, 기존 로직 무손상).
