# ClaudeCode 작업 지시문 — 2순위 (2026-06-26)

> **수신**: Claude Code (노트북, 개발자)
> **발신**: 채팅 Claude (기획자) ← 이주혁 검수 통과
> **선행 상태**: 1순위(보안 마무리) 검수 통과. R23 ✅(강화), 누출 0건. 보안 트랙 안정.
> **이번 범위**: 2순위 — R9(한국 브리핑) + R15(미국 가치주 4기둥 미래성장성★)
> **기준**: 전문적·엄격. 추정 금지. 부분동작을 완성으로 착각 금지. 토스 실주문·연동 금지.

## 2-1. R9 — 한국 브리핑 6항목 (현재 ⬜ → ✅ 목표)

"오늘 한국시장 어때?" 트리거 시 CLAUDE.md 4.2절 명세대로 6항목을 출력하는 기능을 구현한다.

| # | 항목 | 데이터 요건 |
|---|------|-------------|
| 1 | 주요지수&환율 | 코스피, 코스닥, USD/KRW, 외국인 순매수/매도 |
| 2 | 코스피 대장주 | 삼성전자, SK하이닉스, 현대차, 기아, 네이버, 카카오 (현재가+등락%) |
| 3 | 로봇관련주 | 두산로보틱스, 레인보우로보틱스, HD현대로보틱스 동향 |
| 4 | 저평가 가치주 레이더 | PBR/PER 기준 + 근거 |
| 5 | 핵심뉴스 3개 | 증시영향 + 방향성 |
| 6 | 리스크 & 단기전망 | 근거 포함 |

**구현 원칙**
- 데이터는 네이버 API 우선. (이미 검증된 `get_investor_trend`, 일봉, 지수 함수 재사용)
- 수집 실패 시 **R7 원칙 적용**: 해당 항목 GO/출력 보류 + 전용 "수집 실패" 알림.
  (R7의 전용 알림 채널은 아직 🔶 미완이므로, 최소한 "⚠️ ○○ 수집 실패" 표기로 명시할 것.)
- 전문적 어조, 실시간 지표, 근거 필수. 예측사이트 배제.
- 항목 4(저평가 레이더) 판정기준은 무리하게 정교화하지 말 것 — PBR/PER 임계값 + 근거 텍스트
  수준으로 시작. 정밀 기준(PEG·목표가할인율)은 후속(E4)에서.

**보고 시**: 구현 함수명(파일:함수) + **실제 출력 샘플 1회**(실데이터로 6항목 전부 채워진 것).

## 2-2. R15 — 미국 가치주 4기둥 중 미래성장성★ + 펀더멘털 (현재 🔶 → ✅ 근접 목표)

CLAUDE.md 3.1절 4기둥 중 **③미래성장성★(최우선 기둥)**이 비어 있고, ②펀더멘털도 정량 미구현.
현재는 ①밸류에이션·④타이밍만 있음. 나머지 둘을 보강한다.

**③ 미래성장성★ (최우선)**
- 평가 요소: 실적 추세, 가이던스, 신사업, 메가트렌드, 회사 분위기
- ★ 핵심 주의: **정성 항목을 무리하게 수치화하지 말 것.** 억지 점수화는 신뢰도를 떨어뜨린다.
  → "근거 텍스트 + 신뢰도(상/중/하)" 형태로 시작. 데이터로 잡히는 것(실적 추세·가이던스)은
    yfinance/재무 데이터에서, 정성적인 것(분위기·메가트렌드)은 뉴스/검색 근거로.

**② 펀더멘털 보강**
- 지표: 매출·EPS 성장률, 영업이익률, FCF, 부채, 자사주/배당
- yfinance에서 취득 가능한 것부터. 취득 불가 항목은 "(미수집)" 명시 — 추정 금지.

**보고 시**: 구현 위치 + 가치주 1종목(예: NFLX 또는 AMZN)에 대한 **4기둥 전체 평가 샘플 1회**.
- R17(펀더멘털 vs 차트 충돌)이 아직 ⬜이므로, 이번엔 충돌 처리까지는 안 해도 됨.
  단, 4기둥 출력에 차트(타이밍)와 펀더멘털이 같이 보이게 해서 후속 R17의 토대를 만들 것.

## 공통 금지·원칙
- ❌ 토스 API 실주문·연동(R21은 보류). ❌ 키/토큰 평문 하드코딩.
- ❌ 미확인을 ✅로 표기. ❌ 수집 실패 시 GO 생성. ❌ 정성지표 억지 수치화.
- 새 코드가 토큰/키를 출력 경로에 흘리지 않는지 확인(1순위에서 세운 `_redact`/`_tg_post` 원칙 유지).
- 완료 시: 갱신된 R표(R9·R15 행) + self_check 재실행 결과 + 실제 출력 샘플 + 발견된 불일치 보고.
