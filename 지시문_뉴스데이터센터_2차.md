## 배경 (1차 검수 결과 반영)

1차(수집·정규화·저장)는 합격. 단 **핵심 제약 노출**: FMP 뉴스가 현재 플랜에서 402 프리미엄 게이트로 막혀(general/forex/press/stock-search 전부), L1 거시뉴스가 **사실상 네이버 단독**이 됨. 우리 설계의 본질(다소스 교차로 편향 상쇄 + 외신 포함으로 한국 정보 공백 보완 + source_count=신뢰도)이 반쪽이 됨.

**이주혁 확정 방향**: "API 발급은 장벽이 아니다. 최대한 많은 뉴스(원본 데이터)를 쌓아야 시장에서 승산이 있다." → 무료로 긁을 수 있는 뉴스 소스를 최대한 붙인다. 단 무질서한 수집이 아니라 **다소스 → 정규화 → 클러스터링으로 가치화된 자산**으로 만든다.

**저장 철학 (이주혁 확정)**: 원문 전체 저장 안 함. **메인(헤드라인)+요약+결론**까지만. 뉴스는 시장 흐름을 읽는 **보조지표**이지 목적이 아님. 진짜 자산은 "그 뉴스가 무엇을 바라보는가"(시장 하향/상승, 반도체 반짝, 방산 부상 등)를 캐치한 신호. 단 캐치(판단)는 이번 단계 아님 → 이번엔 null 예약만, 채우기는 다음 E5 단계 LLM 판단으로.

## R1. 신규 무료 뉴스 소스 실측 (키 발급 → 실호출 → 판정)

아래 3개 소스를 실제로 키 발급하고 호출해서 판정. **추측 금지, 실측만.** 키는 이주혁이 발급하거나, ClaudeCode가 발급 절차를 안내하고 이주혁이 `.env`에 주입(키 발급·입력·출력은 이주혁만, ClaudeCode는 키 값 보지 않음 — KIS/토스 원칙 동일).

### S6. Finnhub (무인 외신 L1 1순위 후보)
- 무료 API. 시장 뉴스 전용 엔드포인트(market-news) 별도 존재. REST·JSON·표준 HTTP 코드.
- 인증: 토큰 파라미터(`token=apiKey`) 또는 헤더(`X-Finnhub-Token`).
- ★호출 한도: 분당 30콜, 초과 시 429 → 우리 "수집 실패 플래그" 원칙과 연동.
- .env 변수명(제안): `FINNHUB_API_KEY`.
- **실측 항목 4개 (필수 기록): ①발행일 제공 여부 ②매체명 제공 여부 ③실제 호출 한도(무료 티어) ④외신 커버 범위(로이터·블룸버그·CNBC 등 메이저 외신이 실제로 들어오는지).**
- 핵심 검증: **무인 자동화(깃헙 액션)에서 코드가 직접 긁히는가** (web_search와 달리 API라 무인 가능해야 함 — 이게 S6 채택의 핵심 이유).

### S7. Alpha Vantage (sentiment 교차검증 보조)
- 무료 주식 API. JSON·CSV. **글로벌 시장 뉴스 API + 감성 점수(sentiment score) 제공.**
- ★주목: API가 sentiment를 같이 줌 → 우리가 E5에서 만들 sentiment의 **외부 교차검증값**으로 활용 가능(우리 판단 vs Alpha Vantage 판단 대조).
- .env 변수명(제안): `ALPHAVANTAGE_API_KEY`.
- 실측 항목 4개 동일 + ⑤sentiment 점수의 형식·범위(어떤 스케일인지).
- 무료 티어 호출 한도 특히 주의(Alpha Vantage는 분당·일일 제한이 빡빡한 편으로 알려짐 — 실측으로 확인).

### S8. NewsData.io (다소스 폭 확장)
- 무료 티어로 주식·금융 뉴스 제공. 다양한 필터.
- .env 변수명(제안): `NEWSDATA_API_KEY`.
- 실측 항목 4개 동일.

## R2. 기존 소스 + 신규 소스 공통 스키마 정규화 (어댑터)

살아남은(✅/🔶) 소스 전부를 **1차에서 확정한 공통 스키마**로 변환. 소스마다 필드명이 다르므로 어댑터 필수.

### 확정 스키마 (1차 + 2차 신규 필드)
```
{
  "id": "sha1(title+published)[:12]",   // dedup 키
  "layer": "L1_macro | L2_sector | L3_stock",
  "title": "헤드라인 (HTML 태그 제거)",
  "summary": "요약 (HTML 태그 제거)",
  "published": "ISO8601",               // 없으면 null + published_missing:true
  "source": "naver|yfinance|finnhub|alphavantage|newsdata|websearch",
  "publisher": "매체명",                 // 없으면 null + publisher_missing:true
  "url": "원문 URL",
  "symbols": [],                         // 종목 티커 태그
  "sector_tags": [],
  "region": "kr|us|global",
  "attention": null,                     // 네이버 total 등 검색량
  "cluster_id": null,                    // R3에서 부여
  "source_count": 1,                     // 클러스터 내 매체 수 (R3)
  // ── null 예약 (다음 E5 단계에서 LLM 판단으로 채움) ──
  "sentiment": null,                     // 호재/악재/중립
  "score": null,                         // 신뢰도·영향도 점수
  "stance": null,                        // ★신규: 방향성 (상승/하락/중립) — 시장·섹터·종목이 어디로
  "theme": null,                         // ★신규: 주제·테마 태그 (반도체/방산/금리/환율/AI 등)
  "takeaway": null                       // ★신규: "그래서 시장에 뭘 의미하나" 한 줄 캐치 (summary와 구분)
}
```

**★stance/theme/takeaway는 이번 단계 null 예약만.** 채우기 금지(다음 E5 단계 LLM 판단). 단 필드 자리는 이번에 박아둘 것(sentiment·score를 1차에 null 예약했던 것과 동일 패턴).

**summary와 takeaway의 차이 (다음 단계용 메모, 지금 구현 아님):**
- summary = "기사가 뭐라고 썼나" (사실 요약)
- takeaway = "그래서 시장에 뭘 의미하나" (해석 결론)
→ 이번엔 둘 다 자리만. summary는 1차대로 채우고, takeaway는 null.

## R3. 다소스 교차 클러스터링 강화 (★1차 미흡 지점)

1차 검수 지적: 클러스터링이 아직 다소스 교차가 아님(yfinance끼리만 묶임, 계층 다르면 안 묶임). 소스가 늘면 같은 사건이 여러 매체·여러 소스로 중복 유입 → 이걸 묶어야 source_count가 신뢰도 신호로 작동.

### 구현
- **같은 사건 판정**: 제목 유사도(토큰 자카드/코사인 등 stdlib 범위) + 발행시각 근접(예: 24~48h 윈도) → 같은 cluster_id 부여.
- **source_count = 클러스터 내 고유 source/publisher 수.** 여러 매체가 같은 사건 보도 → source_count↑ → "주목도·신뢰도 높은 사건" 신호 (E5 입력값).
- **검증 필수**: 신규 소스 붙인 뒤 **source_count가 실제로 2 이상 나오는 클러스터가 형성되는지** 실측. (1차엔 다소스 교차 클러스터가 안 나왔음 — 이번에 나와야 R3 성공)
- 과설계 금지: 1차처럼 자연 형성 우선. 임계값은 잠정, 사후 조정 대상으로 주석.

## R4. 3계층 자동분류 보정 (L1 외신 보강)

1차 지적: yfinance에 거시·외신 기사(워시 연준의장 등)가 섞여오는데 L3로만 분류됨.

- **거시 키워드·발행처 패턴으로 L1 재분류 규칙 추가**: 연준·금리·CPI·환율·지정학·관세 등 거시 키워드 또는 거시 전문 매체 → symbols 비어도 L1_macro 보조로 승격.
- Finnhub/Alphavantage/NewsData의 거시 기사도 동일 규칙 적용 → **L1이 네이버 단독에서 다소스로 확장**(이번 작업의 핵심 성과 지표).
- L1은 넓게(거시 전반), L2·L3는 보유 우주(반도체·빅테크·AI, US_PORTFOLIO)로 좁게 — 1차 원칙 유지.

## R5. 유인/무인 2모드 분기 (web_search 외신은 보조로)

web_search는 Claude 도구라 **무인 자동화(깃헙 액션)에서 호출 불가.** 그래서 외신 L1의 메인은 **무료 API(Finnhub 등)**가 맡고, web_search는 유인 모드 보강으로 역할 격하.

- **무인 모드(깃헙 액션)**: 네이버 + 무료 API(Finnhub/Alphavantage/NewsData 중 ✅된 것)로 수집. web_search 미사용.
  - 외신 API까지 전부 실패해 외신 0건이면 → 네이버 L1 단독 적재하되 `foreign_source:"unavailable"` 메타 박기. (외신 누락 상태를 데이터에 명시. 보류 아님 — 구조적 미가용이라 마비 회피)
- **유인 모드(채팅/ClaudeCode 개입, web_search 가용)**: 위 + web_search 외신 보강. **외신 수집을 시도했는데 0건이면 → 해당 회차 L1 GO 보류 + "외신 수집 실패" 텔레그램 플래그.** (외신이 돌 수 있는 환경에서 안 들어온 건 진짜 실패 → 엄격 적용)
- web_search 어댑터: 발행일·매체명 추출, 없으면 missing 플래그, 공통 스키마로 정규화.
