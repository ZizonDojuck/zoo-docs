## 1. 이번 작업 범위 (순수 계산 — 외부 호출 최소, 매매판단 로직은 다음 단계)

기술지표 **계산 함수**와 **다중타임프레임 수집·통합**까지. 매수/매도 "판단"은 이번엔 만들지 않음(지표 산출까지만).

### [1] 기술지표 계산 함수 (입력=종가 리스트, 순수 계산)
표준 공식으로 구현. 외부 라이브러리(`ta`, `pandas_ta` 등) 써도 되고 직접 구현해도 됨. 단, **결측·데이터 부족 시 추정 없이 None 반환**.

- `calc_rsi(closes, period=14)` → 최신 RSI 값(0~100). 데이터 < period+1 이면 None.
- `calc_macd(closes, fast=12, slow=26, signal=9)` → dict `{macd, signal, histogram}`. 데이터 부족 시 None.
- `calc_sma(closes, period)` / `calc_ema(closes, period)` — 기존에 이동평균 함수가 이미 있으면 **중복 구현 말고 재사용**. 없으면 추가.
- (선택) `calc_volume_avg(volumes, period=20)` — 거래량 평균(거래량 급증 판정용 기초).

각 함수에 표준 해석 주석:
- RSI: 70↑ 과매수 / 30↓ 과매도 (참고용, 단독 판단 금지)
- MACD: histogram 0선 상향돌파=상승모멘텀 / 하향=하락모멘텀. signal선 교차.

### [2] 다중타임프레임 수집 (분/일/주)
- `get_candles_multi(symbol, market)` — 한 종목의 **분봉·일봉·주봉**을 모아 dict 반환:
  - **분봉(1m)**: `toss_get_candles(symbol, "1m", count=100)` (한국 종목). 미국은 토스 지원 범위 확인 후, 안 되면 yfinance 분봉.
  - **일봉(1d)**: `toss_get_candles(symbol, "1d", count=200)` 또는 기존 네이버/yfinance 일봉. 소스 우선순위는 기존 폴백 규칙 따름.
  - **주봉(1w)**: **토스 미지원** → **일봉을 주 단위로 집계**(매주 시가=주 첫 거래일 시가, 고가=주중 최고, 저가=주중 최저, 종가=주 마지막 거래일 종가, 거래량=주 합계)하여 생성. 별도 함수 `aggregate_daily_to_weekly(daily_candles)`.
  - 각 타임프레임은 수집 실패 시 추정 없이 None + log(R7). 일부 실패해도 나머지는 반환(부분 성공 허용하되 어느 게 실패했는지 표시).

### [3] 타임프레임별 지표 산출 (통합)
- `analyze_timeframes(symbol, market)` — `get_candles_multi` 결과에 [1]의 지표를 적용:
  - 각 타임프레임(분/일/주)별로 RSI, MACD, 주요 이동평균(예: 5·20·60), 현재가 vs 이평 위치를 계산한 dict 반환.
  - **메모리 차트우선순위 참고**(주석으로만, 판단 로직은 다음 단계): 한국단타=거래량>지지저항>추세>모멘텀>캔들 / 미국가치주=주봉추세>52주고가대비>장기지지저항>거래량.
  - 출력은 **지표 숫자 나열까지만**. "매수/매도/보유" 결론은 이번에 만들지 않음(과욕 금지 — 다음 단계에서 사후검증 데이터와 함께 튜닝).

## 2. 안전·보안

- 조회(GET)만, 주문 절대 금지. 토큰·계좌번호 마스킹.
- timeout=10, 실패 시 추정 없이 None+log(R7).
- **self_check 무겁게 금지**: 기존 토큰1+시세1 유지. 다중타임프레임 수집을 self_check에 넣지 말 것(여러 종목×3타임프레임 = 호출 폭증). 지표 계산의 정합성만 **합성 데이터(하드코딩 종가 배열)로 단위검증**.
- 외부 라이브러리 추가 시 `requirements.txt`(있으면)에 반영.

## 4. 검증 후 보고 (맨 끝 1회, 실측)

1. `py_compile` (stock_auto.py, self_check.py)
2. **지표 단위검증(합성 데이터)**: 알려진 종가 배열로 `calc_rsi`/`calc_macd` 계산 → 표준 라이브러리 또는 수계산과 일치하는지(허용오차 내). 데이터 부족 시 None 반환 확인.
3. `aggregate_daily_to_weekly`: 샘플 일봉 배열 → 주봉 집계 정확성(주 OHLCV가 맞게 묶이는가).
4. `analyze_timeframes("005930","KR")` 1회 실행 → 분/일/주 각 RSI·MACD·이평 산출값(실데이터). 어느 타임프레임이라도 수집 실패 시 그 사실 표시.
5. self_check 1회 → 토스 표기·종합등(다중타임프레임 미추가 확인).
6. 보안 스캔 → 노출 0건.

**형식**: 실측값. 지표 숫자는 평문 가능. 토큰·계좌번호 마스킹. 매수/매도 결론은 "이번 미구현(지표 산출까지)" 명시.
