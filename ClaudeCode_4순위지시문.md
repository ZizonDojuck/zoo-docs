# ClaudeCode 작업 지시문 — 4순위 (토스 연동 보안 선행작업, 2026-06-26)

> **수신**: Claude Code (노트북, 개발자)
> **발신**: 채팅 Claude (기획자) ← 이주혁 검수·결정 반영
> **선행 상태**: 3순위(거버넌스·감시종목·로봇주) 검수 통과. RKLB는 처분 확정(아래 정정 참조).
> **이번 범위**: 토스 OpenAPI 연동에 **착수하기 전** 보안 방어막을 먼저 까는 작업.
>            ★ 이번 단계는 토스 키를 코드에 넣지 않는다. 인증·조회·주문 전부 이번 범위 아님.
> **기준**: 전문적·엄격. 추정 금지. 보안 최우선. 토스 실주문·실연동·키 입력 금지.

---

## 왜 이 작업을 먼저 하는가 (배경)
토스 OpenAPI 키는 **계좌를 움직이는 열쇠**다. 과거 텔레그램 봇 토큰이 로그로 샌 사고가
있었고(종결됨), 같은 일이 토스 키로 재발하면 피해가 훨씬 크다. 따라서 토스 키를 코드에
넣기 **전에**, 토스 키가 새지 않도록 막는 방어막(`_redact` 패턴 + `DRY_RUN` 플래그)을
먼저 완성한다. 방어막이 검수 통과해야 다음 단계(토스 인증)로 넘어간다.

---

## 0단계 — RKLB 처분 정정 (먼저, 가벼움)
이전 보고에서 RKLB를 "감시 누락"으로 봤으나, **이주혁이 2026년 6월경 지지선 이탈로
손절 처분**한 것이 확인됐다. 즉 `US_PORTFOLIO`에 RKLB가 없는 현 상태가 **정상**이다.
- CLAUDE.md 부록 A(보유 16종목)에서 RKLB를 제거하거나 "처분됨(2026.06)" 주석을 달아라.
- 코드는 변경 불필요(이미 없음). "정상 확인"으로 보고.

## 1단계 — `_redact`에 토스 키 패턴 추가

현재 `_redact`는 텔레그램 봇 토큰(`bot\d{6,}:[\w-]{30,}`)과 OpenDART 40hex 키만 마스킹한다.
토스 OpenAPI 키/토큰 형식은 다르므로 지금 추가해야 한다.

1-1. 토스 OpenAPI 인증 방식의 키/토큰 형식을 **공식 문서(developers.tossinvest.com)로 확인**하라.
     - OAuth2.0 client_id / client_secret / access_token 각각의 문자열 형식(길이·접두사·문자셋).
     - 추정 금지 — 문서에서 확인한 실제 형식만 반영. 확인 못하면 "(형식 미확인)" 보고 후
       보수적으로 넓은 패턴(긴 영숫자 토큰)으로 임시 마스킹.
1-2. `_redact`에 토스 키/토큰 패턴을 추가하라. 기존 텔레그램·DART 패턴은 유지.
     - access_token, client_secret을 우선 마스킹 대상으로(특히 secret·token).
1-3. 누출 모사 테스트: 가짜 토스 토큰을 포함한 문자열을 `_redact`에 통과시켜
     마스킹되는지 검증하라(텔레그램 때처럼). 결과 보고.

## 2단계 — `DRY_RUN` 플래그 골격

토스 연동 시 실수로라도 실주문이 나가지 않도록, **주문 차단 플래그를 미리 박는다.**

2-1. 전역 설정에 `DRY_RUN = True` (기본값 True) 플래그를 추가하라.
     - `.env`에서 오버라이드 가능하되, **기본값은 반드시 True**(명시 안 하면 차단).
2-2. 향후 토스 주문 함수가 들어갈 자리에 **가드 패턴**을 미리 정의하라(주문 함수 본체는
     아직 만들지 마라 — 가드 골격만):
     ```
     def _order_guard(action):
         if DRY_RUN:
             log(f"[DRY_RUN] 주문 차단됨: {action} (실주문 비활성)")
             return False   # 실행 안 함
         # 실주문은 이주혁 명시적 승인 + 모의투자 검증 후에만 여기 도달
         raise NotImplementedError("실주문은 아직 구현/승인되지 않음")
     ```
     - 이 가드는 "DRY_RUN이면 차단, 아니어도 아직 NotImplementedError"라 **이중 안전**이다.
2-3. `DRY_RUN` 상태를 self_check 대시보드에 표시하라(예: "주문 모드: DRY_RUN(차단)").

## ★ 이번 단계 절대 금지 (재강조)
- ❌ 토스 OpenAPI 키/시크릿을 코드나 .env에 **입력하지 마라** (이번엔 방어막만 만든다).
- ❌ 토스 인증 호출·시세 조회·잔고 조회·거래내역 조회 — **전부 다음 단계**다. 이번엔 안 한다.
