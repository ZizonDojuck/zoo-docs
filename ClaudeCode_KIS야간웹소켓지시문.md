## 1. 작업 전 필독 (웹조사로 확인된 사실)
- **폴더**: `C:\Users\dlwn7\OneDrive\Desktop\주식자동화` → 직행.
- **읽을 것**: KIS 구간(`get_kis_token`, `_kis_get`, `get_kospi_night_futures_kis`, `get_kospi200_front_future_code`), `_redact`, `log`. 반복 통독 금지.
- **확인된 사실**:
  - KIS 야간선물은 **REST 아님 → 실시간 웹소켓**. 국내선물옵션 실시간 11 EP 중 **KRX 야간 5 EP**가 야간선물. **모의 미지원**(실전키라 가능).
  - 웹소켓은 **앱키 대신 "웹소켓 접속키(approval_key)"** 를 별도 발급해 사용. 발급: `POST /oauth2/Approval`, body `{grant_type:"client_credentials", appkey, secretkey}` → `approval_key`.
  - 웹소켓 URL(실전): `ws://ops.koreainvestment.com:21000` (또는 공식 문서 확인). 연결 후 tr_id+tr_key(종목코드)로 구독, 데이터는 `|`,`^` 구분 스트림.
  - 종목코드: 우리가 확정한 **`1A01609`**(또는 야간 전용 코드가 별도면 그것). `get_kospi200_front_future_code()` 활용.
  - 야간선물 실시간 tr_id는 **KRX 야간 선물 체결** 류(정확한 tr_id는 공식 문서/GitHub `domestic_futureoption` 실시간 샘플에서 확인 — 추측 금지).

## 2. 작업: 웹소켓 야간선물 연결 시도

### [2-A] 웹소켓 접속키 발급 `get_kis_ws_approval()`
- `POST /oauth2/Approval` (실전 도메인), body에 appkey/secretkey → `approval_key` 수신.
- 마스킹·실패 시 (None,사유)+log. 접속키도 캐시 가능(단 과발급 주의).

### [2-B] 야간선물 실시간 tr_id 확인
- 공식 문서/GitHub `domestic_futureoption` **실시간(websocket) 샘플**에서 **KRX 야간선물 체결 tr_id**를 확인(예: 국내선물 실시간체결 계열). **추측하지 말고 문서 근거로.**
- 못 찾으면: 후보 tr_id 몇 개를 문서 범위에서만 시도하되, 안 되면 "야간 tr_id 미확정" 정직 보고.

### [2-C] 웹소켓 연결·구독·수신 (단발 테스트)
- `websocket-client` 라이브러리 사용(없으면 설치 `pip install websocket-client`, requirements 반영).
- 웹소켓 연결 → 야간선물 tr_id + 종목코드(`1A01609`) 구독 메시지 전송 → **수 초간 수신** 시도.
- **체결 데이터가 들어오면**: 파싱해서 현재가/시각 추출(`|`,`^` 구분). `get_kospi_night_futures_kis()`가 이 값을 반환하도록 연결.
- **안 들어오면**(연결되나 데이터 없음 / tr_id 거부 / 권한 등): 사유 정직 기록.
- ★ **단발 테스트로**: 무한루프 연결 상주 금지. 수 초 수신 후 닫기(`ws.close()`). 상시 운영 구조는 다음 단계.
