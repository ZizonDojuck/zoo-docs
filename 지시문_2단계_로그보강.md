## 배경 (왜 이 작업인가)

2단계(심층분석 + 판단 로그)는 완료·검수됨. 단 검수에서 **3단계(사후검증·학습 봇)로 가는 연결고리 2개**가 빠진 것이 확인됨. 3단계는 "이 신호가 맞았나 틀렸나"를 라벨링해 학습하는 단계인데, 지금 로그에는 **결과(outcome)를 나중에 붙일 자리가 없음.** 오늘부터 쌓이는 로그가 라벨 자리 없이 쌓이면, 3단계 착수 시 소급 적용이 어려움 → 연료가 반쪽이 됨. 그래서 **지금** 예약 필드를 박아둬야 함.

핵심 원칙 유지: 주문 코드 0줄 / 기록만(판단 반영 안 함) / 3단계 학습 로직은 이번에도 미구현(필드 자리만 예약).

## 작업 항목

### R1. 판단 로그 레코드에 2개 필드 추가 (예약)

`kr_signal_log.jsonl`에 기록하는 레코드(현재 키: ts, code, name, signal, strength, dots, checks, warnings, stop_line, price, ma20, data_fail, kospi_ratio, deep_summary)에 다음 2개를 **추가**:

- `record_id`: 레코드 고유 ID. 권장 = `sha1(ts + code)` 앞 12자리 또는 uuid. 3단계에서 outcome을 이 레코드에 조인하기 위한 키.
- `outcome`: **빈 객체 `{}` 또는 null로 예약.** 지금은 비워둠. 3단계에서 익일/3일/5일 후 주가·적중여부를 나중에 채울 자리. 이번 작업에서 값을 채우는 로직은 만들지 말 것 (자리만 예약).

기존 키는 전부 유지. 순서는 무관하나 record_id를 맨 앞 권장.

### R2. 회피·진입금지 판단도 로그에 전수 기록되는지 확인 + 보장

**가장 중요.** 메모리상 "성공한 회피 판단"(급락장에서 진입금지로 차단한 것)이 3단계의 가장 값진 학습 데이터임. 확인할 것:

- 현재 `job_kr_daytrading`(1단계 스크리닝)에서 로그를 남길 때, **진입 신호가 뜬 종목만** 남기는가, 아니면 **회피·진입금지로 판단된 종목도** 남기는가?
- 만약 진입 신호만 남기고 있다면 → **회피·진입금지 판단도 남기도록 수정.** 스크리닝 대상이 된 모든 종목의 판단 결과(진입/회피/관망 전부)가 로그에 들어가야 함.
- 단, 무한정 다 남기면 로그 폭발 우려 → 스크리닝 후보(동시순매수 등 1차 필터 통과 종목) 범위 내에서 전수 기록. 시장 전체 종목을 다 남기라는 뜻 아님. **"판단 대상이 된 종목은 결과 무관하게 다 기록"**이 기준.
- 확인 결과(현재 어떻게 동작하는지)를 보고에 1줄 명시.
