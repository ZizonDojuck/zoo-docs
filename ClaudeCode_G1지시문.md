## 1단계 — 보안 스캐너에 토스 키 패턴 추가 (★ 키 입력 전에 먼저)

`self_check.py`의 보안 스캐너(§2 평문 키 노출 스캔)는 현재 텔레그램·DART 키만 탐지하고
**토스 키 패턴이 없다.** 이 상태로 토스 키를 넣으면, 실수로 키가 소스에 박혀도 스캐너가
못 잡아 "노출 없음"으로 오판한다. 먼저 막는다.

1-1. `self_check.py` 보안 스캐너에 토스 키 탐지 패턴을 추가하라.
     - G0에서 `_redact`에 넣은 토스 패턴(JWT / Bearer / `c_`·`s_` 접두사 / client_secret·access_token
       키이름 앵커)을 스캐너의 탐지 정규식에도 반영.
     - **단, 생성물(self_check.html/.md)·.env는 스캔 대상에서 제외**(G0 때 세운 원칙 유지 —
       소스 파일만 검사). 스캐너가 자기 자신의 패턴 정의를 오탐하지 않도록 주의.
1-2. 스캐너 검증: 가짜 토스 키를 임시 `.py` 파일에 넣고 스캔 → **탐지(FAIL)** 되는지 확인.
     확인 후 임시 파일 삭제. 정상 소스만 있을 때 오탐 0건도 확인.
1-3. `_redact`(출력 마스킹)와 스캐너(소스 탐지)가 **둘 다** 토스를 커버함을 보고로 명확히.

## 2단계 — 토스 키 자리 준비 (★ 키 입력은 이주혁이 직접)

★★ **ClaudeCode는 토스 키/시크릿을 코드나 .env에 입력하지 마라.** 텔레그램 토큰 때와 동일 —
인증정보는 이주혁이 직접 `.env`에 넣는다. ClaudeCode는 **키를 읽는 자리만 비워두고 로직만** 짠다.

2-1. `.env`에서 토스 키를 읽는 로직을 작성하라(값은 비워둠):
     - `TOSS_CLIENT_ID`, `TOSS_CLIENT_SECRET` 두 키를 환경변수에서 읽는 헬퍼.
     - 키가 없으면 명확한 안내 메시지 + 인증 시도 중단(크래시 금지). 예:
       `"[토스] TOSS_CLIENT_ID/SECRET 미설정 — .env에 입력 필요. 인증 건너뜀."`
     - **이주혁 안내문**: `.env`에 추가할 줄을 README나 주석으로 명시
       (`TOSS_CLIENT_ID=`, `TOSS_CLIENT_SECRET=` — 값은 이주혁이 채움).
2-2. `.gitignore`에 `.env` 등록 재확인(이미 돼 있을 것). 토스 키가 들어갈 것이므로 재점검.

## 3단계 — OAuth 토큰 발급만 테스트 (조회 0건)

G0에서 확인한 토스 인증 방식: **OAuth2.0 Client Credentials Grant, `POST /oauth2/token`**.

3-1. 토큰 발급 함수를 작성하라(`get_toss_token` 등):
     - client_id/secret으로 `POST /oauth2/token` 호출 → access_token(JWT) + expires_in 수신.
     - 응답·예외 로그는 반드시 `_redact` 경유(토큰이 로그에 새지 않게). `_tg_post` 패턴 준용.
     - 토큰 캐싱(만료 전 재사용) 골격까지만. 과설계 금지.
3-2. **이번 단계에서 호출하는 것은 토큰 발급 엔드포인트 단 하나.**
     - ❌ 시세·잔고·거래내역·주문 엔드포인트 호출 금지(전부 G2 이후).
     - 키가 .env에 있으면: 실제 토큰 발급 1회 테스트 → 성공/실패 + (토큰은 마스킹된 형태로만) 보고.
     - 키가 없으면(이주혁 미입력 시): 발급 로직이 "미설정 안내 후 안전 중단"하는지만 확인하고
       "키 입력 후 재실행 필요"로 보고. **추정으로 성공 단정 금지.**
3-3. self_check에 토스 상태 표시 갱신:
     - 토스 = `N/A(미인증)` → 키 입력+발급 성공 시 `인증 OK(토큰 발급)`, 실패 시 `인증 실패`로.
     - 단 **조회/주문은 여전히 미구현**임을 명시(인증≠조회 가능).
